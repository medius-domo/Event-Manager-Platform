Environment Variables Check

Instructions:

In Vercel, make sure you have these environment variables set:

  • VITE_SUPABASE_URL - Your Supabase project URL
  • VITE_SUPABASE_ANON_KEY - Your Supabase anonymous key

Important: After adding or changing environment variables in Vercel, you MUST redeploy your project!

Build-time Check:

This page was built and deployed. If you can see this, the deployment succeeded.

Next Steps:

  1. Go to your Vercel project settings
  2. Navigate to Environment Variables
  3. Verify both VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY exist
  4. Redeploy the project from the Deployments tab
  5. Check the browser console for error messages

Common Issues:

  • Environment variables must start with VITE_ to be accessible in the frontend
  • Changes to environment variables require a new deployment
  • Make sure there are no spaces before/after the values
  • The Supabase URL should start with https://
